# AI 面試模擬器專案簡介

## 專案願景
建立一個 AI 驅動的面試模擬系統，幫助求職者練習技術面試，提供即時反饋和程式碼分析。

## 核心需求

### 主要功能需求
1. **AI 面試官對話**: 使用 Gemini LLM 進行自然語言對話
2. **程式碼分析**: 支援多種程式語言的語法檢查和品質評估
3. **面試類型支援**: 技術面試、行為面試、系統設計面試
4. **即時反饋**: 提供面試表現評分和改進建議

### 技術需求
- **後端**: Python Flask 框架
- **LLM 整合**: Google Gemini API
- **程式碼驗證**: clang++ 和多語言語法檢查
- **API 設計**: RESTful 介面
- **前端**: 簡潔的 Web 界面

### 非功能性需求
- **效能**: 低延遲回應 (<3秒)
- **可靠性**: 穩定的 API 服務
- **擴展性**: 模組化架構支援未來功能擴展
- **安全性**: 程式碼安全檢查和輸入驗證

## MVP 範圍限制
- **簡化版本**: 專注核心功能驗證
- **3週時程**: 快速原型開發
- **單用戶**: 不支援多用戶和持久化
- **基礎UI**: 最小可用介面

## 成功標準
1. 能夠進行基本的 AI 面試對話
2. 支援至少 3 種程式語言的程式碼分析
3. 提供準確的語法檢查和建議
4. 完整的 API 文檔和測試覆蓋
5. 可擴展的架構基礎
